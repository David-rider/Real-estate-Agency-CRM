# Developer Guide (开发者指南)

Welcome to the Real Estate Brokerage Management Platform Agent CRM project! This document outlines the core architecture, data models, frameworks, and the exact workflow required to modify or extend the system in the future.

## 🏗️ 1. Tech Stack & Architecture (技术栈与架构)

This is a modern Full-Stack application separated into two main folders: `frontend` and `backend`.

### Frontend
- **Framework:** Next.js (App Router, React 19)
- **Language:** TypeScript
- **Styling:** Tailwind CSS + Radix UI / Headless UI patterns
- **State Management:** React Context (for `AuthContext` and `I18nContext`)
- **Location:** `/frontend`
- **Start Command:** `npm run dev` (Runs on `http://localhost:3000`)

### Backend
- **Framework:** Node.js + Express
- **Language:** TypeScript
- **ORM (Database):** Prisma
- **Database Engine:** PostgreSQL
- **Authentication:** JWT (JSON Web Tokens) & `bcryptjs`
- **Location:** `/backend`
- **Start Command:** `npm run dev` (Runs on `http://localhost:4000`)

## 📂 2. Core Directory Structure (核心目录结构)

If you need to find a specific feature, look here:

### Backend (`/backend`)
*   `prisma/`
    *   `schema.prisma`: **🔥 THE MOST IMPORTANT FILE.** This defines every database table and relation.
    *   `seed.ts`: Mock data generator to seed the database with test users, properties, and deals.

### Frontend (`/frontend`)
*   `src/app/`: The Next.js App Router. Each folder represents a URL route (e.g., `/login`, `/dashboard`, `/crm`).
    *   `layout.tsx`: The global wrapper containing Sidebar, Header, and Providers.

*   `src/lib/`:
    *   `i18n/translations.ts`: **🔥 CRITICAL DICTIONARY.** Contains all English, Chinese, and Spanish text.
    *   `auth/AuthContext.tsx`: Manages the current logged-in user state.

## 📊 3. Database Models (数据库模型大全)

All current models are strictly defined in `backend/prisma/schema.prisma`. 

1. **`User`**: The system users (Agents, Managers, Admins). It holds the email, hashed password, `role` (ADMIN, MANAGER, AGENT), and subscription `tier`.
2. **`Client`**: The CRM contacts tied to a specific `User`. It tracks whether they are a BUYER, SELLER, or REFERRAL.
3. **`Property`**: The real estate listings tracking price, beds, baths, and status (e.g., ACTIVE, IN_CONTRACT).
4. **`Transaction`**: The Deal pipeline tracking status (OFFER_REVIEW, CLEARED_TO_CLOSE). It acts as a junction linking a `User`, a `Client`, and a `Property`.
5. **`Commission` (Upcoming)**: To track waterfall split calculations and agent net payouts.

## 🚀 4. How to Build a New Feature (如何开发新模块：标准工作流)

If you need to add a completely new page or feature, **ALWAYS** follow this exact sequence to conform to our architectural standards:

### Step 1: Database (如果需要存新数据)
1. Open `backend/prisma/schema.prisma`.
2. Add your new `model` and define its relations.
3. Run `npx prisma db push` in the backend terminal to sync the PostgreSQL database.
4. (Optional) Update `backend/prisma/seed.ts` to add mock data and run `npx tsx prisma/seed.ts`.

### Step 2: Backend API (构建数据接口)
1. Create a new file in `backend/src/routes/` (e.g., `feature.ts`).
2. Write your Express routes (`GET`, `POST`, `PUT`, `DELETE`).
3. If it requires login, wrap the route with `authenticateToken`.
4. Register the new router in `backend/src/server.ts`.

### Step 3: Internationalization / i18n (多语言支持)
1. Open `frontend/src/lib/i18n/translations.ts`.
2. Find the English (`en`), Chinese (`zh`), and Spanish (`es`) objects.
3. Add a new block for your feature (e.g., `newFeature: { title: "Hello", ... }`) in ALL THREE languages. **Never hardcode English directly into the React components.**

### Step 4: Frontend UI (搭建视图)
1. Create a new folder in `frontend/src/app/` (e.g., `new-feature/page.tsx`).
2. Import `useI18n()` to get the `t` object for text rendering.
3. Import `useAuth()` if you need the user's `token` or `role`.
4. Use standard `fetch()` locally (or `axios`) to point to your new `http://localhost:4000/api/new-feature` endpoint, ensuring you pass the `'Authorization': \`Bearer ${token}\`` header.
